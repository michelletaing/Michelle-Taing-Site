OOTD: Digital Closet Application

Jan - Apr 2023

View your organized closet

Discover the wonders of your well-arranged wardrobe, neatly divided into five distinct categories. Simply click on each category to unveil a collection of clothing items of that particular type.

Populate your closet

Upload an image of a clothing piece, name it, and place it into a category. After a clothing item has been added to your closet, you can easily update any of these fields.

Build an outfit

Swipe through your clothing items to quickly visualize your outfit through our outfit builder interface. Name your outfit and save it -- you can come back later and edit the outfit if you wish!

Team Structure

This application was created collaboratively with three other peers. We split off into two development teams: frontend and backend. Although I was formally on the frontend team, I also did backend development for select features.

Inspiration

We decided to create a closet application because it is a tool we would all use. The idea is simple, practical, and applicable to anyone who owns clothing. Additionally, the features we wanted to implement were complex enough to allow us to create a substantial application. Simply put, we were all looking for a project that would ease us into the full-stack development world. Creating this application involved a lot of “firsts” for everyone — this was our first time creating a full-stack application, and we were using completely new tools/technologies.

Timeline and Workflow

This project was split into 4 distinct sprints over the course of ~3 months. The requirements for each sprint were variable, but generally, we would implement X number of features (which we defined at the start of the sprint), and each newly implemented feature was supported by unit tests (backend) or end-to-end tests (frontend). Documentation for each sprint can be found here.

Personal Involvement

Generally, my role was focused on integrating the frontend interface with the backend API. Oftentimes, I created basic interfaces using Angular Material components, integrated the features with our API, and tested for functionality. I would hand this off to my frontend partner who would improve the UI. I also worked closely with the backend team to define the structure and logic of the API functions. Collaborating closely with both sides of development was extremely fulfilling and definitely my favorite part about working on this project!

I could share so much more about my experience on this project — what it was like learning new technologies on the fly, what I learned about effective teamwork/collaboration, the joys and struggles of working on this project, etc. — but I will keep it to this for now. :)
